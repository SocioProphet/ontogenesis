# Semantic Enterprise v0.1 Release Note

## Status

Semantic Enterprise v0.1 is stable as a governed Ontogenesis tranche.

This release line integrates semantic-enterprise architecture, lifecycle vocabulary, external-standard alignment scaffolds, sector modules, scenario fixtures, SPARQL query examples, named-graph governance, downstream import contracts, and import-contract validation.

## Scope

Semantic Enterprise v0.1 includes:

- lifecycle and governance middle modules
- semantic mapping and named graph governance vocabulary
- alignment scaffolds for gist, FIBO, UCO/CASE, D3FEND, IOF SCO, and DoDAF/CCO
- sector modules for finance, threat intelligence, investigation, supply chain, and defense/C2
- scenario fixtures for AML/KYC, CTI lifecycle, custody timeline, supply-chain resilience, and common operational picture usage
- read-only SPARQL examples for each scenario
- named-graph governance fixture for sector scenario graphs
- downstream import bridge contract for platform consumers
- semantic import-contract validation workflow

## Rollup registry

The v0.1 rollup registry is:

- `catalog/semantic_enterprise_v0_1_registry.ttl`

It imports the base registry and the semantic-enterprise supplement registries:

- `catalog/semantic_enterprise_registry.ttl`
- `catalog/semantic_sector_registry.ttl`
- `catalog/semantic_sector_scenarios_registry.ttl`
- `catalog/semantic_query_import_registry.ttl`

## Validation posture

The tranche is validated by the existing Ontogenesis gates:

- RDF parse validation
- SHACL gates
- JSON-LD roundtrip checks
- dist build and ledger verification
- SBOM generation
- no-raster-images check

It is also guarded by:

- `.github/workflows/semantic-import-contract.yml`
- `scripts/validate_semantic_import_contract.py`

## Design posture

Ontogenesis remains the canonical authored source. Downstream systems may compile, index, materialize, query, or display the semantic-enterprise assets, but they must preserve source path, canonical IRI, registry entry, validation gate, and named graph governance metadata where available.

The external ontology strategy is governed alignment, not bulk vendoring by default.

## Downstream consumers

The v0.1 import bridge names the following consumers:

- Prophet Platform
- Sherlock Search
- AgentPlane
- PolicyFabric
- SourceOS
- DeliveryExcellence

## Non-goals

This tranche does not define production graph storage, live ingestion, operational response playbooks, incident-response procedures, or access-control implementation. Those belong to downstream runtime systems and must remain separately governed.

## Next tranche

The next tranche should add small importer/runtime smoke tests that compile the v0.1 rollup registry into downstream-readable manifests and assert traceability for each scenario, query, named graph, and SHACL gate.
