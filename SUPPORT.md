# Support

## Getting help

- For usage questions and implementation guidance, open a GitHub Discussion or issue in this repository.
- For bugs, open a GitHub issue with clear reproduction steps and expected vs. actual behavior.

## Security contact

For private vulnerability reporting, use GitHub private vulnerability reporting for this repository if enabled, or contact repository maintainers through private channels.

## What to include in support requests

- repository version/commit
- files/modules involved
- commands run and output
- minimal reproduction data
